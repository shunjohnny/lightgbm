🌟 アルゴリズムの基本ポイント

機械学習の3大タスク

大きく分けて3つあるの！

  • 回帰：数値を予測！（例：売上予測、株価予測）
  • 分類：カテゴリを予測！（例：スパム判定、画像認識）
  • クラスタリング：グループ分け！（例：顧客セグメンテーション）

📊 線形回帰の詳細

線形回帰の数式

予測値 = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ
w₀
バイアス項
w₁~wₙ
重み
x₁~xₙ
特徴量

正則化付き線形回帰

  • Ridge回帰：L2正則化で過学習を防ぐ
  • Lasso回帰：L1正則化で特徴量を選択
  • Elastic Net：L1とL2両方使う！

線形回帰の注意点

  • 外れ値に弱い！
  • 特徴量間の相関が強いとダメ！
  • 非線形な関係は表現できない！

🌳 決定木アルゴリズム詳説

決定木の仕組み

木構造を使ってデータを分割していくの！

  • ノード：条件分岐のポイント
  • リーフ：最終的な予測値
  • 分岐基準：不純度（ジニ係数/エントロピー）を減らす！

決定木のメリット

  • 理解しやすい！
  • 前処理少なくても大丈夫！
  • 非線形な関係も表現できる！

🚀 アンサンブル学習の種類

バギング（Bagging）

データをランダムサンプリングして並列学習！

  • ランダムフォレスト：代表的な手法
  • 特徴量もランダムに選ぶ
  • 過学習しにくい！

ブースティング（Boosting）

逐次的に学習して、弱点を補強！

  • 勾配ブースティング：残差を学習
  • AdaBoost：誤分類データの重みを上げる
  • 精度は高いけど、過学習しやすい

📚 ライブラリ詳細解説

scikit-learn

from sklearn.ensemble import GradientBoostingRegressor model = GradientBoostingRegressor( n_estimators=100, # 木の数 learning_rate=0.1, # 学習率 max_depth=3 # 木の深さ )

LightGBM

import lightgbm as lgb model = lgb.LGBMRegressor( objective='regression', num_leaves=31, learning_rate=0.1, n_estimators=100 )

💡 ハイパーパラメータ調整のコツ

パラメータ 意味 調整のコツ
learning_rate 学習率 小さいほど安定するけど時間かかる
n_estimators 木の数 多いほど複雑なパターンが学習できる
max_depth 木の深さ 深いほど過学習しやすい

調整の基本戦略

  1. learning_rateを小さめに固定（0.01-0.1）
  2. max_depthを調整（3-10程度）
  3. n_estimatorsを増やして様子見
  4. early_stoppingでオーバーフィット防止

🔍 モデル評価指標

回帰タスクの評価指標

  • MAE：平均絶対誤差
  • MSE：平均二乗誤差
  • RMSE：二乗平均平方根誤差
  • R2：決定係数

分類タスクの評価指標

  • Accuracy：正解率
  • Precision：適合率
  • Recall：再現率
  • F1-score：PrecisionとRecallの調和平均
